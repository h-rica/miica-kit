# MEMORY

Durable project memory for coding agents and maintainers.

## Purpose

Capture stable lessons that should affect future work:

- recurring blockers
- durable workflow defaults
- guardrails that prevent repeated mistakes
- source-of-truth notes that frequently drift

Do not use this file for temporary task status, ad hoc TODO lists, or speculative notes.

## How to use

- Read this file at the start of every non-trivial task.
- Update it only when a lesson is likely to matter again.
- Prefer short, concrete rules over long narrative logs.
- Remove stale guidance when code or process reality changes.

## Suggested sections

### Source of truth

- stack and framework versions that matter
- files or manifests that win when docs drift

### Workflow defaults

- preferred commands
- verification bar
- release or QA expectations

### Recurring blockers

- known failure modes
- environment gotchas
- setup steps people forget

### Guardrails

- directories or files to treat carefully
- boundaries between subsystems
- rules for safe edits
