# phase-plan-ceo-review

Use this skill to challenge the request before implementation.

## Goal

Find the highest-leverage version of the request and make the scope decision explicit.

## Scope modes

- Expansion
- Selective Expansion
- Hold Scope
- Reduction

## Workflow

1. Challenge whether the named feature is actually the feature.
2. Reframe around user outcomes.
3. Present scope options and tradeoffs.
4. Lock the chosen direction before coding.

## Outputs

- clarified problem statement
- chosen scope mode
- success criteria
- deferred ideas list

## Guardrails

- do not accept a narrow implementation framing too quickly
- do not write code in this phase
- keep time and ambition tradeoffs explicit

## Exit condition

After producing the outputs for this skill:

- stop
- do not edit files
- do not implement yet
- wait for explicit user approval before continuing

If the same user message explicitly asks for CEO review plus implementation, deliver the CEO review output first and only then continue.
