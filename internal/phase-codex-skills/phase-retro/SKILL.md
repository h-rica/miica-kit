# phase-retro

Use this skill for evidence-based retrospectives.

## Goal

Summarize what actually happened and what should change next.

## Evidence sources

- commits and PRs
- shipped changes
- QA reports
- incidents or regressions
- test and coverage trends

## Outputs

- throughput summary
- quality and test trends
- review and QA health
- growth opportunities

## Guardrails

- use evidence, not vibes
- show trends when historical data exists
- identify what improved, what regressed, and what should change in process

## Exit condition

After producing the retrospective:

- stop
- do not edit files in this skill
- wait for explicit user approval before continuing
