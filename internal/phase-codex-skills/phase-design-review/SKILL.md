# phase-design-review

Use this skill to audit and fix a live UI.

## Goal

Inspect the built experience in a real browser and directly fix the most important design problems.

## Workflow

1. Inspect the UI in a real browser.
2. Score the experience across the same dimensions as planning design review.
3. Fix the highest-leverage issues directly.
4. Re-check the result visually.

## Outputs

- visual audit
- prioritized UI issues
- implemented visual fixes
- before/after evidence when useful

## Guardrails

- preserve the existing design system when one exists
- avoid subjective nitpicks with no user impact
