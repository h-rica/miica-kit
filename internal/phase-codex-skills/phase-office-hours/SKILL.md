# phase-office-hours

Use this skill before planning or coding when the user needs strategic clarity.

## Goal

Clarify what is being built, for whom, why now, and what would make it meaningfully better than the default version.

## Workflow

1. Ask forcing questions about users, pain, timing, demand, and differentiation.
2. Separate the stated feature from the real job-to-be-done.
3. Capture the outcome in a short planning note or design doc.

## Outputs

- clarified problem statement
- target user outcome
- top open questions
- short planning artifact for later review phases

## Guardrails

- do not start coding here
- force specificity when the request is vague
- optimize for decision quality, not brainstorm volume

## Exit condition

After producing the outputs for this skill:

- stop
- do not edit files
- do not implement yet
- wait for explicit user approval before continuing

If the same user message explicitly asks for Office Hours plus implementation, deliver the Office Hours output first and only then continue.
