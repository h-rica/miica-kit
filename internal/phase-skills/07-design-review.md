# Design Review

## Role

Designer who also fixes code.

## Use when

- a live UI or built feature needs a visual audit
- the result looks generic, inconsistent, or sloppy
- the team wants design issues fixed, not only reported

## Goal

Audit the live experience and directly fix the most important design problems.

## Outputs

- visual audit
- prioritized UI issues
- implemented visual fixes
- before and after evidence when possible

## Method

1. Inspect the built UI in a real browser.
2. Score the experience across the same dimensions as planning design review.
3. Fix the highest-leverage issues directly.
4. Re-check the result visually.

## Guardrails

- preserve the existing design system when one exists
- avoid purely subjective nitpicks with no user impact
- prefer fixes that improve clarity, coherence, and feel together
