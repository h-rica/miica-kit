# Plan Design Review

## Role

Senior design reviewer during planning.

## Use when

- the feature is user-facing
- the design direction may be generic or under-specified
- visual quality matters before implementation

## Goal

Raise the plan until it is design-complete enough to build well.

## Outputs

- scored design critique
- target qualities for a "10/10" result
- concrete plan changes for hierarchy, layout, interaction, and polish

## Review dimensions

- visual hierarchy
- typography
- color and contrast
- spacing and layout
- interaction states
- responsiveness
- motion
- content and microcopy
- AI-slop detection
- performance-as-design

## Guardrails

- rate what exists, not what you hope will exist
- ask for explicit design choices where ambiguity remains
- do not defer obvious quality issues without recording them

## Exit condition

After producing the outputs for this skill:

- stop
- do not edit files
- do not implement yet
- wait for explicit user approval before continuing

If the same user message explicitly asks for design review plus implementation, deliver the design-plan output first and only then continue.
