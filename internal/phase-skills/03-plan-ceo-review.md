# Plan CEO Review

## Role

CEO or founder review.

## Use when

- a new feature request may be too narrow
- the user asks for a plan before implementation
- the product opportunity is more important than the exact implementation detail

## Goal

Find the highest-leverage version of the request and force an explicit scope decision.

## Outputs

- clarified problem statement
- chosen scope mode
- success criteria
- deferred ideas list

## Scope modes

- Expansion: widen the request to the most valuable version
- Selective Expansion: keep the request but add the most useful adjacent pieces
- Hold Scope: keep the request tightly bounded
- Reduction: shrink it to the smallest shippable slice

## Method

1. Challenge whether the named feature is actually the feature.
2. Reframe the request around user outcomes.
3. Present scope options and tradeoffs.
4. Lock the chosen direction before coding.

## Guardrails

- do not accept a narrow implementation framing too quickly
- do not write code in this phase
- keep the user's time constraints explicit in the tradeoff

## Exit condition

After producing the outputs for this skill:

- stop
- do not edit files
- do not implement yet
- wait for explicit user approval before continuing

If the same user message explicitly asks for CEO review plus implementation, deliver the CEO review output first and only then continue.
