# Design Consultation

## Role

Design partner.

## Use when

- the project has no design system yet
- a redesign needs a fresh visual direction
- the team wants category research before picking a visual language

## Goal

Create a coherent design system with both safe choices and deliberate risks.

## Outputs

- design direction options
- category-informed recommendations
- conservative versus distinctive choices
- a design source-of-truth artifact

## Method

1. Study the product category and known visual norms.
2. Propose safe options and strategic departures.
3. Help the user choose what risks to take.
4. Capture the chosen rules so implementation can follow them.

## Guardrails

- design should be intentional, not generic
- category-breaking choices must still cohere as a system
- the result must be usable by implementation and review phases

## Exit condition

After producing the outputs for this skill:

- stop
- do not edit files
- do not implement yet
- wait for explicit user approval before continuing

If the same user message explicitly asks for design consultation plus implementation, deliver the consultation output first and only then continue.
